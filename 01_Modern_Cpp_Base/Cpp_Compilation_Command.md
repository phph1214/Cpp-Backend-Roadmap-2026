# 编译 (Compile)
不要只敲 g++ main.cpp，那是学生气的写法。我们要加上调试信息、开启警告并指定标准。

```Bash
g++ main.cpp -o main -g -Wall -std=c++17
```

**指令拆解：**
* g++: 调用 GNU C++ 编译器。

* main.cpp: 你的源文件。

* -o main: Output（输出）。指定生成的可执行文件名叫 main。如果不写这个，默认会生成一个叫 a.out 的文件，非常不专业。

* -g: Generate Debug Info。必须加！ 否则稍后教你用 GDB 调试时，你什么都看不见。

* -Wall: Warning All。开启所有常见警告。零警告 (Zero Warnings) 是大厂代码提交的硬性标准。如果编译出警告，视同编译失败。

* -std=c++17: 强制使用 C++17 标准。防止你写出过时的 C++98 代码。

# 2. 运行 (Run)
编译成功后（没有报错），执行以下命令运行程序：

```Bash
./main
```
**为什么要加 ./？**

* Linux 出于安全考虑，默认不会在当前目录查找可执行文件。

* . 代表“当前目录”。

* / 是路径分隔符。

* ./main 的意思是：“执行当前目录下的 main 文件”。

⚠️ 硬核面试考点：按下 g++ 发生了什么？

当你敲下那行 g++ 命令时，系统并没有一步到位。面试官常问：“C++ 从源码到可执行文件经历了哪四个阶段？”

* 预处理 (Preprocessing): 处理以 # 开头的指令（如 #include 展开，#define 替换）。

* 编译 (Compilation): 把 C++ 代码翻译成汇编代码 (.s 文件)。

* 汇编 (Assembly): 把汇编代码翻译成机器码（目标文件 .o）。

* 链接 (Linking): 把你的 .o 文件和 C++ 标准库（如 iostream）的 .o 文件打包在一起，生成最终的 main 可执行文件。
